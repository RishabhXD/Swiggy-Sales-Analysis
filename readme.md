# Swiggy Sales Analysis Dashboard 📊

A data analytics project focused on analyzing Swiggy food delivery sales data using Python. This project uncovers business insights related to revenue trends, customer behavior, restaurant performance, and operational efficiency through data visualization and exploratory data analysis (EDA).

---

# 🚀 Project Overview

The goal of this project is to perform end-to-end sales analysis on Swiggy transactional data and build insightful visualizations to support data-driven business decisions.

The dashboard analyzes:
- Monthly & Quarterly Sales Trends
- Weekly & Daily Order Patterns
- Top Performing Cities & Restaurants
- Customer Ratings & Satisfaction
- Payment Method Analysis
- Order Status Insights

---

# 📂 Dataset Information

The dataset contains food delivery transaction records with attributes such as:

- Order Date
- Price (INR)
- City
- State
- Dish Name
- Restaurant Name
- Rating
- Payment Method
- Order Status

# 📈 Key Visualizations

## Monthly Sales Trend
Analyzed revenue growth patterns across different months.

## Daily & Weekly Trend Analysis
Identified peak ordering days and weekly demand fluctuations.

## Quarterly Performance Summary
Compared quarterly sales, order volume, and customer ratings.

## Top Cities by Sales
Highlighted highest revenue-generating cities.

## Restaurant Performance Analysis
Identified top-performing restaurants based on total sales.

## Customer Ratings Distribution
Analyzed customer satisfaction trends.

## Payment Method Analysis
Studied customer payment preferences.

## Order Status Analysis
Monitored completed, cancelled, and pending orders.

# 🔍 Key Insights

- Certain cities contributed significantly higher revenue.

- Customer ratings indicated generally positive service satisfaction.
